P6Log

P6Log is an open source application included in the P6Spy distribution that intercepts and logs the database statements of any application that uses JDBC. This application is particularly useful for developers to monitor the SQL statements produced by EJB servers, enabling the developer to write code that achieves maximum efficiency on the server.  P6Log is designed to be installed in minutes, and can be put to work immediately.

P6Spy consists of a number of modules that provide different functionality, dependent on your needs. The P6Log module is enabled by default. You disable/enable the P6Log module by editing the spy.properties configuration file. If you comment out the module, it is not loaded, and the functionality is not available. If it is not commented out, then the functionality is available. Here is an example:

#################################################################
# MODULES #
# #
# Modules provide the P6Spy functionality. If a module, such #
# as module_log is commented out, that functionality will not #
# be available. If it is not commented out (if it is active), #
# the functionality will be active. #
# #
# Values set in Modules cannot be reloaded using the #
# reloadproperties variable. Once they are loaded, they remain #
# in memory until the application is restarted. #
# #
#################################################################

module.log=com.p6spy.engine.logging.P6LogSpyDriver
#module.outage=com.p6spy.engine.outage.P6OutageSpyDriver

P6Log shares some common property file settings that let you specify which tables you want to log, the file and location of the log file, if you want to show the stacktrace (where the JDBC statement is being executed), and more. See the common property file settings for details.
